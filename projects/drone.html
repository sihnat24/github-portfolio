Overview

What it is & why

Detecting objects in drone footage is a surprisingly hard problem: targets are tiny, scenes are cluttered, and a missed detection in a real surveillance context matters more than a false alarm. The goal was to build a system that finds the best model configuration automatically, without needing a human to babysit every experiment. The project is structured as a constrained experiment: hyperparameter search runs on 10% of the VisDrone2019-DET dataset locally, with the most promising configurations graduating to full training on a dedicated GPU. VisDrone2019-DET is a public benchmark of aerial footage covering 10 object categories including pedestrians, cars, and cyclists filmed across dense urban environments. The training loop is inspired by Andrej Karpathy's autoresearch concept, where an AI agent iterates on its own code and hyperparameters to drive performance forward with minimal human input.

Technical Approach

Architecture & decisions

The core of the project is an automated training loop built with Claude Code, where an AI agent iterates on the model's hyperparameters, runs training, scores the result, and decides whether to keep or discard the change. Performance is measured using a custom scoring metric that combines recall and precision, deliberately weighted 3:1 in favor of recall, reflecting the real-world cost of missing an object versus flagging a false positive. If a run improves the score, the change is committed to git as a permanent checkpoint. If it does not, the repository is reset to the previous state and the loop tries a different direction. This means the codebase only ever moves forward on real gains, and the full history of what worked and what did not is preserved in the commit log. Each experiment runs on 10% of the dataset locally, making the iteration cycle fast enough to run dozens of configurations in a single session, with the strongest setups graduating to full training on a dedicated GPU.

Results

Outcomes & metrics

Across 20 experiments on the VisDrone2019-DET benchmark, the agentic loop improved the composite score (0.75 × recall + 0.25 × precision) from 0.0983 to 0.1133, a +15.3% gain, with recall rising from 0.075 to 0.089 and precision from 0.170 to 0.188, all within a fixed 10-epoch / 10%-data budget.

The agentic loop automated the full experiment cycle: modifying hyperparameters, running training, scoring the result, and committing or reverting the change based on performance. This made it practical to run 20 controlled experiments with consistent tracking, something that would be error-prone and tedious to manage manually. One concrete finding that emerged from this volume of trials is that data augmentation consistently hurt performance at this training scale, a result that would have been difficult to confirm without repeated controlled comparisons.

The loss curves (Figure 1) indicate the model is undertrained rather than mis-configured. Box loss, classification loss, and focal loss are all still decreasing with no sign of plateauing, suggesting that additional training epochs would continue to drive improvement.

Per-class analysis reveals meaningful variation in model behavior. Most classes follow a typical precision-recall tradeoff curve (Figure 2), but car is a notable exception: its curve rises before falling, indicating more stable detection across confidence thresholds. The per-class bar chart (Figure 3) confirms that car scores the highest overall and is the only class where recall exceeds precision. At the other end, awning-tricycle and bicycle score the lowest, likely due to their small apparent size in aerial imagery and relative underrepresentation in the 10% data fraction used for training.

Low detection confidence across validation images further supports the undertrained diagnosis. All experiments were run on an Apple M4, which constrained runs to 10 epochs at 10% of the full dataset. The project is now migrating to an NVIDIA RTX 3060 to remove that constraint and allow for substantially longer training runs.

The next step is to rerun the experiment suite at 50 epochs, using the best-performing 10-epoch configuration as the starting baseline. With a larger training budget, we expect per-class performance to improve broadly and for augmentation to transition from harmful to beneficial.
